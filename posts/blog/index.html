使用Hugo + Github Pages部署博客

 2020.10.3  Blog  874
使用的操作系统: Arch Linux
需要git操作基础

  • 安装Hugo

    ​ Hugo官方的定义是: Hugo is a fast and modern static site generator written in Go, and designed to make website creation fun again.

    ​ Arch Linux的官方的软件源里有hugo。

    sudo pacman -S hugo

​ 其他发行版也应该能有包管理器安装，比如Debian系的apt， RHEL系的yum，实在不行可以上官网下载然后阅读文档安装。

  • 创建博客目录

    ​ Hugo项目就是一个博客。例如，在用户目录(~)下创建名为Blog的项目:

    hugo new site Blog

    ​ 创建完成后，在 Blog文件夹下会生成以下文件结构：

    .
├── archetypes  # 存放生成博客的模版
├── assets      # 存放被 Hugo Pipes 处理的文件
├── config      # 存放 hugo 配置文件 支持 JSON YAML TOML 三种格式配置文件
├── content     # 存放 markdown 文件
├── data        # 存放 Hugo 处理的数据
├── layouts     # 存放布局文件
├── static      # 存放静态文件 图片 CSS JS文件
└── themes      # 存放主题

  • 下载主题并创建第一篇博客文章

    ​ 在Blog文件夹， themes就是存放主题的位置，你可以浏览Hugo官方主题网站选择你喜欢的主题，并按照说明文档将主题从github/gitlab下载到本地。比如:

    git clone https://github.com/reuixiy/hugo-theme-meme themes/meme

    ​ 克隆操作后面的url就是主题的github地址，下载好就能开始写博客了。

    ​ Blog下的config.toml是整个博客的配置文件，比如博客的主题，标题等等。将 config.toml 替换为示例配置。

    mv config.toml config.toml.bak && cp themes/meme/config-examples/zh-cn/config.toml config.toml

    ​ 新建一篇文章和一个关于页面：

    hugo new posts/hello-world.md
hugo new about/_index.md

    ​ 在Blog/content/posts/hello-world.md文件中以markdown的格式写上内容， 最后在本地运行Hugo:

    hugo server

    ​ Hugo会默认选用localhost:1313这个端口，在浏览器中查看效果时输入地址: localhost:1313。

  • 部署到Github Pages

    ​ 创建一个github远程仓库， 名字格式为[your name].github.io, 比如 link.github.io， 并在本地生成好ssh密钥对， 以便于上传数据。

    ​ 在Blog下执行hugo (直接输入hugo) ， 生成成功后， 会多出一个名为public的文件夹， 此文件夹就是我们要上传到github远程仓库的所有文件。接下来:

    git init
git add . && git commit -m 'first commit' && git branch -M master
git remote add git@github.com:xxx/xxx.github. io
git push -u origin
