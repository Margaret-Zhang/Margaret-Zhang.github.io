GNU/Linux安装MySQL-8.0.20通用二进制离线包（以systemd管理）

注意：以下命令均由超级管理员（root）执行

  1. 将安装包移动到自定义的位置（推荐/usr/local）

    cd /usr/local
tar -xf /path/to/mysql-8.0.20-linux-glibc2.12-x86_64.tar.xz
mv mysql-8.0.20-linux-glibc2.12-x86_64 mysql

    groupadd mysql
useradd -r -g mysql -s /bin/false mysql
cd /usr/local/mysql
mkdir mysql-files
chown mysql:mysql mysql-files
chmod 750 mysql-files
   
#初始化mysqld服务，生成初始密码（如果需要用自定义配置，可以在加上--defaults-file选项）
bin/mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data
   
#开启SSL
bin/mysql_ssl_rsa_setup
   
#启动mysqld服务
support-files/mysql.server start
   
#使用初始密码登录mysql
bin/mysql -u root -p
   
#在mysql当中修改root密码
# ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '<new password>';

    #创建mysql日志文件目录
mkdir /var/log/mysql
touch /var/log/mysql/mysql.log
chown -R mysql:mysql /var/log/mysql
   
mkdir /var/lib/mysql
chmod 755 /var/lib/mysql
mkdir /var/run/mysql
chown -R mysql:mysql /var/run/mysql
   
#修改mysql.server的basedir和datadir
vim /usr/local/mysql/support-files/mysql.server
   
#创建mysqld.service文件用于systemd管理
cat > /usr/lib/systemd/system/mysqld.service << "EOF"
[Unit]
Description=MySQL Server
Documentation=man:mysqld(8)
Documentation=http://dev.mysql.com/doc/refman/en/using-systemd.html
After=network.target
After=syslog.target
   
[Install]
WantedBy=multi-user.target
   
[Service]
User=mysql
Group=mysql
   
# Have mysqld write its state to the systemd notify socket
Type=forking
   
# Disable service start and stop timeout logic of systemd for mysqld service.
TimeoutSec=0
   
# Start main service
ExecStart=/usr/local/mysql/bin/mysqld --daemonize
   
# Use this to switch malloc implementation
EnvironmentFile=-/etc/sysconfig/mysql
   
# Sets open_files_limit
LimitNOFILE=10000
   
Restart=on-failure
   
RestartPreventExitStatus=1
   
# Set environment variable MYSQLD_PARENT_PID. This is required for restart.
Environment=MYSQLD_PARENT_PID=1
   
PrivateTmp=false
EOF
chmod 644 /usr/lib/systemd/system/mysqld.service
   
#开机启动
systemctl enable mysqld

解决动态链接库问题

​ 实测Arch Linux在安装过程中，进入MySQL控制台失败，错误提示“libtinfo.so.5动态链接库不存在”。解决方法是archlinuxcn仓库中有ncurse5-compat-libs软件包，安装之后正常进入MySQL控制台。

Microsoft Windows平台安装MySQL-8.0.20的步骤请移步记录仓库。
